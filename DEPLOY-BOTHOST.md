# Развёртывание бота на хостинге (по форме из панели)

Бот собирается из Git-репозитория и работает на long polling — веб-домен и webhook НЕ нужны.

## Заполнение формы создания бота

| Поле | Значение |
|------|----------|
| **Название бота** | Global Metrostroi — синхронизация |
| **Платформа** | Telegram |
| **Язык разработки** | Python |
| **Версия Python** | 3.11 (рекомендуется) |
| **Bot Token** | токен от @BotFather |
| **Локация развёртывания** | Нидерланды (Амстердам) — Telegram заблокирован в РФ |
| **Git URL репозитория** | ссылка на ваш репозиторий с папкой bot/ |
| **Ветка** | main |
| **Доп. настройки → Главный файл** | bot.py |
| **Использовать домен** | НЕ включать (long polling) |
| **Собственный Dockerfile** | можно включить — в папке bot/ уже есть Dockerfile |

> Важно: если репозиторий содержит весь проект, а не только папку bot/,
> укажите главный файл как `bot/bot.py` или залейте содержимое bot/ в отдельный репозиторий.

## Переменные окружения (блок «Переменные окружения»)

Добавьте по кнопке «+ Добавить переменную» (или «Загрузить из .env»):

```
TG_BOT_TOKEN=<токен бота>
TG_ADMIN_IDS=<ваш telegram id>
EXPORT_CHAT_ID=<chat_id для авто-выгрузки, можно пусто>
EXPORT_INTERVAL_MIN=60
DB_HOST=<хост MySQL на REG.RU>
DB_PORT=3306
DB_DATABASE=<имя БД>
DB_USERNAME=<пользователь БД>
DB_PASSWORD=<пароль БД>
```

## ВАЖНО: доступ бота к БД

Бот работает в Нидерландах, а БД — на REG.RU. Поэтому `DB_HOST=localhost` НЕ подойдёт.
Нужно разрешить удалённые подключения к MySQL в панели REG.RU
(«Удалённый доступ к MySQL» / добавить IP хостинга бота в разрешённые),
а в `DB_HOST` указать внешний хост/IP сервера БД REG.RU.
Если удалённый доступ включить нельзя — запускайте бота на том же сервере, где БД.

## Команды бота

- `/stats` — сводка по таблицам
- `/export` — выгрузка всех таблиц в Excel (+ лист «Права доступа» и логи)
